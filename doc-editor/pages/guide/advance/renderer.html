渲染及性能优化

Viewfly 采用自上而下的单向数据流方案，当父组件变化时，子组件也会自动渲染，同时，Viewfly 也会跳过数据没有数据变更的分支。总体来说，Viewfly 目前的 diff 算法是非常高效的。大部分场景我们并不需要关心性能问题。但总有一些场景，数据量特别大，变动又特别频繁。我们可以参考下面的策略进行性能调优。

分支调优

我们可以拆分变动频繁的部分和不常变动的部分为两个组件，并且让两个组件不属于父子或父孙关系，当 Viewfly 在更新 DOM 时，就会跳过没有变化的分支，这可以显著的提高 diff 的效率。

可参考如下 diff 更新流程：

Key

对于列表渲染，我们可以通过给循环生成的节点添加 key 的方式来“优化”性能。相同 key 的元素会直接复用，而不是重新创建或删除。key 的值只可以是字符串字数字，且在相同层级内必须是唯一的。

list.map(item => {
  return <li key={item.id}>xxxx</li>
})
Jsx
冷知识：添加 key 只是会让 diff 算法强制重用 DOM 节点，在很多情况下，添加 key 不一定会使性能更优。

withMemo()

对于部分组件，我们明确知道什么时候应该更新，我们可以使用 withMemo 函数，明确告知 Viewfly，可以跳过组件的更新。

withMemo 的第一个参数为一个函数，当组件需要更新时，Viewfly 会调用此函数，并传入当前的组件的 Props 和上一次的 Props。你可以通过返回 true 来让 Viewfly 跳过本次更新。

import { withMemo } from '@viewfly/core'

function List(props) {
  return withMemo((currentProps, prevProps) => {
    // 判断组件是否可重用，返回 true 表示重用，则不会再 diff 此组件及子组件
    return currentProps.value === prevProps.value
  }, () => {
    return (
      <li>{props.value}</li>
    )
  })
}

状态内置

Viewfly 的更新策略是组件依赖的 Signal 数据发生了变化。在大型列表中，我们可以把外部状态，改变为内部状态，这将使性能得到成倍的提升。

在以下的表格中，我们会高亮用户选中的行。

优化前，我们在 Table 组件渲染时使用了 selected，这将导致 selected 的值发生变更时，整个 Table 组件将再次渲染。

import { useSignal } from '@viewfly/core'

const list = useSignal(Array.from({length: 100000}).map(() => {
  return {
    id: Math.random(),
    name: Math.random().toString(16),
    age: 32,
    address: 'xxx'
  }
}))

const selected = useSignal(null)

function Row(props) {
  return () => (
    <tr class={{higtlight: props.selected}}>
      <td>{props.name}</td>
      <td>{props.age}</td>
      <td>{props.address}</td>
      <td><button onClick={function() {
        selected.set(props.id)
      }}>选择</button>
    </tr>
  )
}

function Table() {
  return () => (
    <table>
      {
        list().map(item => {
          return <Row {...item} selected={item.id === selected()}/>
        })
      }
    </table>
  )
}
Jsx

优化后，我们使用 useDerived 勾子监听了 selected Signal，并根据当前组件的 props.id 判断当前组件是否选中。由于 useDerived 返回的 Signal 的值是根据传入的回调函数计算结果决定的，所以，当我们的选中的行变更时，那些选中状态不产生变更的 Row 组件将不会有任何渲染，一次切换最多只会引起两个 Row 组件的变更 diff。

import { useSignal, useDerived } from '@viewfly/core'

const list = useSignal(Array.from({length: 100000}).map(() => {
  return {
    id: Math.random(),
    name: Math.random().toString(16),
    age: 32,
    address: 'xxx'
  }
}))

const selected = useSignal(null)

function Row(props) {
  const isSelected = useDerived(() => {
    return props.id === selected()
  })
  return () => (
    <tr class={{higtlight: isSelected()}}>
      <td>{props.name}</td>
      <td>{props.age}</td>
      <td>{props.address}</td>
      <td><button onClick={function() {
        selected.set(props.id)
      }}>选择</button>
    </tr>
  )
}

function Table() {
  return () => (
    <table>
      {
        list().map(item => {
          return <Row {...item}/>
        })
      }
    </table>
  )
}
